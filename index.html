1

核心问题：Deals 是真需求还是伪需求？

用户的真实心理

我想旅行，但我想花得更值。如果有一个机会让我觉得"赚了"，我会更果断下单。

这不是"我要找便宜"，而是"我需要一个理由让自己觉得这个消费是正当的"。这是真实存在的决策心理。

需求真伪判断

层次 需求描述 判断
"我想花得值" 消费决策中普遍存在的心理需求 真需求
"我想要一个地方专门看打折" 有则逛逛，没有也不会主动找 弱需求
"一个填满折扣商品的常驻频道" 供给和信任都难以持续支撑 大概率伪需求

关键发现：其他场域的缺位

Trip.com 现有场域（首页、行程页、搜索结果）的推荐逻辑都是个性化推荐——"猜你喜欢什么"。没有任何场域从价格视角帮用户做决策。

用户在整个 Trip.com 生态里缺少一个声音："你现在买这个，是划算的。"

这就是 Deals 存在的合理性所在。

3

Deals 的独占价值（没有独占价格时）

前提条件

Deals 频道的商品不是专享价格——用户在搜索页也能以同样价格购买。因此，Deals 独占的不是价格本身，而是价格判断 + 时机信号。

1. 价格变动可见性

告诉用户"什么东西正在变便宜"——某酒店降了30%、某航线本周比上周低、某目的地进入淡季拐点。其他场域不提供这个信息。

2. 反个性化视角

用户本来没想去清迈，但看到机酒 ¥1,200，产生了新的出行意愿。Deals 能用价格创造新需求，而非只满足已有偏好。

3. 时机判断服务

帮用户回答"现在是不是买的好时机"——价格趋势、供给变化、季节规律。这是购买决策中最难自行获取的信息。

4. 主动猎取心态

个性化推荐是被动接受（系统找你），Deals 是主动猎取（用户来找机会）。两种心态转化路径完全不同。

4

选品策略：从"价格事件"出发

❌ 错误做法

用户是谁 → 推什么货

和其他场域的推荐逻辑一样，只是加了划线价。本质没有区别。

✓ 正确做法

发生了什么价格事件 → 哪些用户会感兴趣

以价格信号为起点选品，真正区别于个性化推荐。

价格事件类型

事件类型 具体场景 适合模块
供给驱动 航司放特价舱位、酒店尾房清仓、淡季降价 限时闪购
价差驱动 比历史价/市场价显著便宜的商品 超值发现
组合驱动 机+酒组合比单买省超过 20% 打包省更多
时机驱动 目的地进入价格低谷、平台补贴活动 当季低价目的地

选品红线

  • 绝不放没有价格故事的商品 — 无法说明"为什么现在值得买"的商品不进入 Deals
  • 绝不放无法验证优惠的商品 — 没有对比锚点（历史价/市场价），用户无法判断是否真的"Deal"
  • 宁可少，不凑数 — 用"少"建立信任，每一个出现的商品都应该有足够的价格说服力
  • 品质兜底 — 评分低于阈值的商品不进入 Deals，保护频道信任

供给侧筛选框架：什么商品有资格进入 Deals

核心原则

"值得买"不是一个主观判断，而是可以从供给数据中客观计算出来的。关键问题是：用什么数据证明"现在比平时更值"，然后把这个证据翻译成用户能秒懂的语言。

通用准入条件（所有品类）

不管什么品类，一个商品要进入 Deals 频道，必须满足至少一个可量化的"值得买"条件：

条件类型 判断逻辑 用户感知翻译
历史低价 当前价 < 近30/60/90天均价 × 阈值 "近X天最低价"
同类价差 当前价 < 同城市/同星级/同航线均价 "比同类便宜 X%"
限时降价 供应商主动设置了促销价 + 截止时间 "限时特价，还剩 X 天"
组合价差 打包价 < 各单品分别购买总价 "比单买省 ¥X"
稀缺供给 低价库存剩余 < 阈值 "仅剩 X 间/X 个席位"

不满足以上任何一条的商品 = 没有资格进入 Deals。这是铁律，不因运营需要填充内容而放松。

"值得买"传达公式与供给优先级

信息传达公式

不管什么品类，向用户传达"值得买"都应该包含四个要素：

值得买 = 价格证据 + 降价理由 + 品质保证 + 行动理由

缺少任何一个要素，用户的转化链条就会断裂

要素 回答的问题 示例 缺失后果
价格证据 便宜多少？跟什么比？ "比近30天均价低 35%" 用户不相信便宜
降价理由 为什么现在便宜？ "淡季尾房 / 航司闪促 / 平台补贴" 用户怀疑是套路
品质保证 便宜是因为差吗？ "★4.8 / 含早 / 直飞 / 免费取消" 用户担心踩坑
行动理由 为什么不能等？ "还剩2天 / 仅剩3间 / 近期在涨" 用户想"以后再说"

供给优先级分层

不是所有满足条件的商品权重相同。需要按"值得买"的强度排序：

第一梯队：强价格事件 — 必推
  • 航司官方闪促（有截止时间，稀缺性真实）
  • 酒店集团大促（万豪/希尔顿/IHG 会员日等）
  • 历史最低价突破（算法自动监测到的新低）
  • 平台独家补贴（用户在别处买不到这个价）
第二梯队：可量化优势 — 选推
  • 比近30天均价低 25%+
  • 组合比单买省 20%+
  • 同城同级最低价
  • 去年同期价格更低 20%+
第三梯队：弱信号 — 谨慎推（宁可不放）
  • 仅比均价低 10-15%（优势不够明显，用户无感）
  • 降价理由不明确（无法解释为什么便宜）
  • 品质信号不足（评分低/评价少/新品无口碑）

原则：第三梯队的商品，宁可不放进 Deals，也不要稀释频道的"值得买"浓度。频道的信任来自于"每一个出现的商品都经得起追问"，而不是数量。

心智传递的三层结构

第一层：入口心智 — 让用户相信"值得逛"

  • 聚合性数字："今日 326 个限时折扣"
  • 最大折扣预告："低至 2 折起"
  • 社会证明："过去 1 小时 1,200 人抢购"

第二层：浏览心智 — 让用户确认"真的便宜"

  • 每个商品必须有价格锚点（原价划线、历史低价对比、省多少）
  • 折扣理由透明化（为什么便宜：尾房、预售、平台补贴、航司促销）
  • 品质保证信号（评分、星级、已售量）— 打消"便宜没好货"的顾虑

第三层：行动心智 — 让用户感到"不能等"

  • 倒计时 / 剩余库存百分比
  • "XX 人正在看" 实时热度
  • 价格趋势图："今天是近 30 天最低"

风险与红线

!
做成"个性化推荐 + 划线价"

和其他场域无本质区别，频道失去存在理由。必须坚持"价格事件驱动"的选品逻辑。

!
用虚假原价/伪紧迫感凑数

用户一旦发现"Deals 里的东西搜一下也是这个价且没有时间限制"，信任崩塌不可逆。

!
放入第三梯队弱信号商品凑数

10%的价格优势用户无感知，反而拉低整个频道的"值得买"浓度和可信度。

供给不足时强行填满频道

宁可展示"今天只有 8 个值得买的 Deal"，也不要用低质量商品破坏频道心智。

⚠
退化为运营资源位

判断标准：如果把频道入口去掉，是用户投诉还是运营投诉？后者说明它已不是用户产品。

总结

Deals 频道的本质

Deals 的本质不是"另一个推荐场"，而是"价格事件的翻译器"。

它把用户自己很难获取的信息——什么在降价、什么是历史低、什么组合更划算——转化为明确的购买信号。

这是其他所有以"个性化推荐"为核心逻辑的场域无法提供的独占价值。

选品原则

从"价格事件"出发，而非从"用户画像"出发。没有价格故事的商品不进入频道。每个商品必须通过决策树的四层校验。

信息传递原则

不是"推荐给你"，而是"发生了这件事"。用客观的价格证据说话。每个卡片必须包含：价格证据 + 降价理由 + 品质保证 + 行动理由。

信任原则

宁可少，不凑数。只推第一、第二梯队的供给。频道里每一个商品都应该经得起"真的比平时便宜吗？"的追问。

差异化原则

时刻自问：如果去掉价格信号，这个页面和首页推荐有什么区别？如果没有——方向错了。Deals 的每个设计决策都应该服务于"价格视角"。
