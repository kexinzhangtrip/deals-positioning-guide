1

核心问题：Deals 是真需求还是伪需求？

用户的真实心理

我想旅行，但我想花得更值。如果有一个机会让我觉得"赚了"，我会更果断下单。

这不是"我要找便宜"，而是"我需要一个理由让自己觉得这个消费是正当的"。

需求真伪判断

层次需求描述判断
"我想花得值"消费决策中普遍存在的心理需求真需求
"我想要一个地方专门看打折"有则逛逛，没有也不会主动找弱需求
"一个填满折扣商品的常驻频道"供给和信任都难以持续支撑大概率伪需求

关键发现：其他场域的缺位

Trip.com 现有场域的推荐逻辑都是个性化推荐——"猜你喜欢什么"。没有任何场域从价格视角帮用户做决策。

用户缺少一个声音："你现在买这个，是划算的。" 这就是 Deals 存在的合理性。

3

Deals 的独占价值（没有独占价格时）

Deals 频道的商品不是专享价格——用户在搜索页也能以同样价格购买。独占的不是价格本身，而是价格判断 + 时机信号。

1. 价格变动可见性

告诉用户"什么东西正在变便宜"——某酒店降了30%、某航线本周比上周低。

2. 反个性化视角

用户本来没想去清迈，但看到机酒 ¥1,200，产生了新的出行意愿。用价格创造新需求。

3. 时机判断服务

帮用户回答"现在是不是买的好时机"——价格趋势、供给变化、季节规律。

4. 主动猎取心态

个性化推荐是被动接受（系统找你），Deals 是主动猎取（用户来找机会）。

4

选品策略：从"价格事件"出发

❌ 错误做法

用户是谁 → 推什么货

和其他场域一样，只是加了划线价。

✓ 正确做法

发生了什么价格事件 → 哪些用户会感兴趣

以价格信号为起点选品。

价格事件类型

事件类型具体场景适合模块
供给驱动航司放特价舱位、酒店尾房清仓、淡季降价限时闪购
价差驱动比历史价/市场价显著便宜的商品超值发现
组合驱动机+酒组合比单买省超过 20%打包省更多
时机驱动目的地进入价格低谷、平台补贴活动当季低价目的地

选品红线

  • 绝不放没有价格故事的商品 — 无法说明"为什么现在值得买"的不进入
  • 绝不放无法验证优惠的商品 — 没有对比锚点，用户无法判断
  • 宁可少，不凑数 — 每一个出现的商品都应该有足够的价格说服力
  • 品质兜底 — 评分低于阈值的商品不进入 Deals

供给侧筛选框架：什么商品有资格进入 Deals

"值得买"不是主观判断，而是可以从供给数据中客观计算出来的。

通用准入条件

条件类型判断逻辑用户感知翻译
历史低价当前价 < 近30/60/90天均价 × 阈值"近X天最低价"
同类价差当前价 < 同城市/同星级/同航线均价"比同类便宜 X%"
限时降价供应商主动设置了促销价 + 截止时间"限时特价，还剩 X 天"
组合价差打包价 < 各单品分别购买总价"比单买省 ¥X"
稀缺供给低价库存剩余 < 阈值"仅剩 X 间/X 个席位"

不满足以上任何一条的商品 = 没有资格进入 Deals。铁律，不因运营需要而放松。

7

"值得买"传达公式与供给优先级

值得买 = 价格证据 + 降价理由 + 品质保证 + 行动理由

缺少任何一个要素，用户的转化链条就会断裂

要素回答的问题示例缺失后果
价格证据便宜多少？跟什么比？"比近30天均价低 35%"用户不相信便宜
降价理由为什么现在便宜？"淡季尾房 / 航司闪促"用户怀疑是套路
品质保证便宜是因为差吗？"★4.8 / 直飞 / 免费取消"用户担心踩坑
行动理由为什么不能等？"还剩2天 / 仅剩3间"用户想"以后再说"

供给优先级分层

第一梯队：强价格事件 — 必推
  • 航司官方闪促 / 酒店集团大促 / 历史最低价突破 / 平台独家补贴
第二梯队：可量化优势 — 选推
  • 比近30天均价低 25%+ / 组合比单买省 20%+ / 同城同级最低价
第三梯队：弱信号 — 谨慎推（宁可不放）
  • 仅比均价低 10-15% / 降价理由不明确 / 品质信号不足

心智传递的三层结构

第一层：入口心智 — "值得逛"

  • 聚合数字："今日 326 个限时折扣"
  • 最大折扣预告："低至 2 折起"
  • 社会证明："过去 1 小时 1,200 人抢购"

第二层：浏览心智 — "真的便宜"

  • 每个商品必须有价格锚点（原价划线、历史低价对比）
  • 折扣理由透明化（尾房、预售、平台补贴）
  • 品质保证信号（评分、星级、已售量）

第三层：行动心智 — "不能等"

  • 倒计时 / 剩余库存百分比
  • "XX 人正在看" 实时热度
  • 价格趋势图："今天是近 30 天最低"

行中：Deals 如何服务正在旅行的用户

行中用户的心理不同于行前——他们已经花了钱、已经在目的地，关注的不是"要不要出发"而是"怎么让这趟旅程更值"。

三层框架在行中的变体：闪购层对行中无效（不会冲动买不相关目的地的机票），核心价值在发现层和个人层。

层级行前逻辑行中变体
发现层"现在去哪最划算""你在的地方，现在做什么最划算"
个人层"你关注的降价了""你的行程里，还能省的地方"

行中发现层：当地价格洼地

  • "今天这个景点门票 ¥89（平时 ¥150）"——淡日票价差
  • "酒店加购晚餐 ¥128 双人（餐厅单买 ¥400+）"——酒店内 upsell 折扣
  • "明天下雨，室内景点今天票价最低"——天气驱动的供需变化
  • "你附近 500m 有个体验在做闪促"——LBS + 价格事件

行中个人层：行程省钱优化

  • "你后天的酒店降了 ¥200，要免费改价吗？"——已有订单价保
  • "你的行程还没有门票，联票比单买省 ¥95"——行程缺口 + 组合价差
  • "回程机票比你买时降了，下次可以用锁价功能"——建立回访认知

行中推荐的三个关键差异

维度行前行中
地理范围全球什么便宜当前位置附近什么便宜
时间窗口未来几个月今天/明天
品类权重机票酒店为主门票/体验/餐饮/交通为主

核心逻辑不变：仍然需要通过决策树（价格证据→降价理由→品质→紧迫性），只是输入数据变了。

风险与红线

!
做成"个性化推荐 + 划线价"

和其他场域无本质区别，频道失去存在理由。

!
用虚假原价/伪紧迫感凑数

用户一旦发现"搜一下也是这个价"，信任崩塌不可逆。

!
放入第三梯队弱信号商品凑数

10%的价格优势用户无感知，拉低频道可信度。

⚠
供给不足时强行填满频道

宁可展示"今天只有 8 个值得买的 Deal"。

⚠
退化为运营资源位

判断：去掉入口后是用户投诉还是运营投诉？

总结

Deals 的本质不是"另一个推荐场"，而是"价格事件的翻译器"。

它把用户很难获取的信息——什么在降价、什么是历史低、什么组合更划算——转化为明确的购买信号。这是其他所有"个性化推荐"场域无法提供的独占价值。

核心矛盾：方案在两个方向之间摇摆

PRD v3 实质上在做什么

PRD v3 构建了一个 六层页面架构（L1-L6）× 人群矩阵（5段）× 旅行阶段（7期）= 超级个性化推荐频道。

这恰好是定位指南中标注的最大风险：做成"个性化推荐 + 划线价"，与首页/行程页无本质区别。

PRD v3 的逻辑

用户是谁 × 阶段 → 推什么货

核心竞争力 = 个性化精准度

定位指南的逻辑

发生了什么价格事件 → 谁受益

核心竞争力 = 价格判断的可信度

判断标准：如果把所有价格标签去掉，PRD v3 的页面和首页推荐有什么区别？如果答案是"更精准的推荐"——方向可能错了。
1

L2 "黑盒主战场"：方向对了，执行需修正

PRD：L2 定位为"实时 engagement slot"，服务 58% 无标签黑盒用户，做实时推荐。

建议：做成"今日最强价格信号"

  • 当天降幅最大的 3-5 个 Deal（跨品类）
  • 即将到期的限时促销（紧迫性最高）
  • 刚突破历史低价的商品（新鲜感最强）

"东京机票今天比上周低 40%"——不管你是家庭用户还是商旅用户，都是有效信息。

2

6 层 × 5 人群 × 7 阶段 = 大量内容填充需求。必然引入弱信号商品。L6 点击率仅 4.2%，层级太深用户已不看。

L1 标题层 + L4 矩阵层 + L5 商品流 + L6 IP内容
→ 合并或降级为辅助模块
人群标签不应重建整个页面，而是调整三层内的排序权重。家庭用户的"发现层"里亲子酒店排前面，但页面结构完全一样。

"人群×阶段"矩阵：运营成本被严重低估

一句话判断

PRD v3 在把 Deals 做成"更聪明的推荐频道"——但真正的机会是做"Trip.com 里唯一提供价格视角的地方"。

建议把核心驱动力从"人群精准度"切换为"价格事件强度"。

7

价格事件驱动如何确保人群精准匹配

"价格事件驱动" ≠ 不做人群匹配。关键区别是匹配发生在哪一层——不是选品入口，而是排序和触达层。

核心逻辑：先筛质量，再匹配人群

STEP 1: 价格事件池（入口）
  • 全品类扫描，找出今日所有满足准入条件的商品
  • 判断标准：有没有价格事件？
    └─ 没有 → 不进入 Deals
    └─ 有 → 进入候选池

STEP 2: 人群匹配（排序层）
  • 从候选池中，根据用户画像/行为排序
  • 排序信号：
    ├─ 搜索历史：搜过东京 → 东京降价排前面
    ├─ 行为偏好：总看 4-5 星 → 经济型酒店排后面
    ├─ 行程相关：3 月有曼谷行程 → 曼谷门票优先
    └─ 家庭标签：有亲子标签 → 亲子酒店排前面

STEP 3: 展示（传达层）
  • 每个商品都有真实的价格故事
  • 人群只决定顺序，不决定是否出现

与纯人群驱动的对比

维度纯人群驱动（PRD v3）价格事件 + 人群排序（建议）

底线保障无 — 可能推没折扣的商品凑数有 — 候选池里只有真实优惠

关键洞察：价格事件自带人群匹配

很多价格事件天然锁定了目标人群，不需要额外匹配：

价格事件天然匹配的人群排序补充
亲子乐园门票 5 折家庭用户（自带）有孩子的用户排前
商务舱特价商旅用户（自带）近期搜过商务舱的排前
清迈机酒 ¥1200有东南亚偏好的用户搜过清迈/泰国的排前
酒店尾房 ¥299价格敏感型用户低星偏好 + 近期浏览排前
3月东京机票破历史低所有用户（跨人群有效）搜过日本 > 搜过亚洲 > 全部

结论：大部分精准匹配通过"排序"就能实现。只有少数场景（如"个人层—你关注的降价了"）需要强个性化，而那部分天然有行为数据。

实施框架：三级匹配机制

第一级：行为信号（强匹配）

用户搜/看/收藏过的目的地、酒店、航线降价了

→ 进入"个人层 For You" · 推送/通知触达 · 最高优先级

第二级：画像信号（弱匹配）

用户属于某人群，该人群偏好的品类出现价格事件

→ 在"发现层 Discover"中排序靠前 · 不影响选品入池

第三级：无信号/冷启动（通用）

按价格事件强度排序 —— 降幅最大、最稀缺的排前面

→ 对所有用户有效 · 不依赖人群标签 · 解决 58% 黑盒用户

形象比喻：PRD v3 是"超市按人群分区，每个区摆不同商品"；建议方案是"超市按当日特价分区，但每个人进门时拿到一张个性化的'今日推荐路线图'"——路线不同，但货架上的每样东西都确实在打折。
